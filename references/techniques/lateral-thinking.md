# Lateral Thinking

## When to Use
Creative block. You've been going in circles. Every idea feels like a variation of the same thing. The team has been staring at the whiteboard for an hour with nothing new.

## The Method
1. State the problem as currently understood.
2. Deliberately abandon the most obvious approach path.
3. Enter through a side door: pick a random object, concept, or domain.
4. Force a connection: "How would a [random thing] solve this problem?"
5. Extract the principle from the forced connection, then refine it into something practical.

## Example
Problem: "How do we reduce customer support tickets?"
Obvious path: Better docs, better UI, FAQs. All variations of "explain better."
Side door: "How does a forest handle too many animals drinking from the same stream?" → Multiple streams form. Animals drink at different times. Some get water from plants.
Extracted ideas: Shard support across specialized agents by issue type. Stagger onboarding so users don't all hit the same confusing step at once. Embed answers in the product itself (in-context), not in a separate docs site.

## Why It Works
The brain's associative network fires along well-worn paths. Random entry points force new neural routes, creating genuinely novel connections.
