# Provocation (PO)

## When to Use
Status quo thinking. "We've always done it this way." Entrenched assumptions. Industry-wide blind spots.

## The Method
1. Identify a foundational assumption in the current approach.
2. Make a provocative statement that contradicts it: "PO: [absurd reversal of the assumption]."
3. Treat the PO seriously. What would the world look like if it were true?
4. Extract useful movement from the provocation — not the literal PO, but where it points.

## Example
Assumption: "Software needs a user interface."
PO: "Software should have NO user interface."
Movement: What if software was purely API-driven and everything happened via natural language? What if there were no buttons, no menus, no forms? → Chat interfaces, voice agents, ambient computing, zero-UI design.

## Why It Works
De Bono's PO ("Provocative Operation") breaks pattern-thinking. The absurd statement jolts the brain out of its usual tracks, opening space for genuinely new ideas.
