# Inversion

## When to Use
Strategy, risk analysis, planning, decision-making. Especially powerful when you're trying to achieve a positive outcome.

## The Method
1. Define the goal clearly.
2. Instead of asking "How do I achieve this?", ask "What would guarantee failure?"
3. List everything that would cause catastrophic failure.
4. Now invert each failure cause into a prevention strategy.
5. The result is a list of what to avoid — often more actionable than a list of what to pursue.

## Example
Goal: "Make this project successful."
Inversion: "What would make this project fail catastrophically?" → Unclear scope, no user feedback for 6 months, building features nobody asked for, team of one with no code review, no backup if key person leaves.
Prevention: Lock scope in week 1, ship to users weekly, only build requested features, hire/partner before scaling, document everything.

## Why It Works
Avoidance is psychologically easier than pursuit. You're better at spotting dangers than opportunities — leverage that.
