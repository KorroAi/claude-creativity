# What If Scenarios

## When to Use
Vision, roadmap, disruption planning. Long-term thinking. Preparing for scenarios others haven't considered.

## The Method
1. State the current reality as a baseline.
2. Ask "What if [major change] happened tomorrow?"
3. Explore the second and third-order effects. Don't stop at the obvious.
4. Identify opportunities and threats in the new world.
5. Work backward: what should you do today to be positioned for that future?

## Example
Baseline: We build mobile apps like everyone else.
What if: "What if mobile phones disappeared tomorrow and were replaced by AR glasses?"
1st order: No more touch interfaces. Everything is spatial.
2nd order: Apps don't exist — experiences exist at locations. An "app" is just a layer on reality.
3rd order: The concept of "screen time" dies. Computing becomes ambient.
Action today: Design our architecture to be UI-agnostic. Separate logic from presentation so we can target any interface.

## Why It Works
Counterfactual thinking is how humans plan. "What if" scenarios let you rehearse futures without committing to them, revealing robust strategies that survive multiple possible worlds.
